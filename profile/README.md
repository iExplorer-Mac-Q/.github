## 📄 Overview

<a href="#iExplorer">iExplorer</a> is an iOS device file management application for Mac that provides direct access to iPhone and iPad file contents — browsing, transferring, and managing files on connected iOS devices without the restrictions that iTunes and Finder impose on device file access.

The <a href="#iExplorer">direct music library transfer</a> copies music and playlists from iPhone to Mac without iTunes sync — transferring individual songs, complete albums, or entire playlists from the device to the Mac's music library or any folder. This is the reverse direction that iTunes and Finder don't support: moving music from the phone to the Mac rather than only from Mac to phone. Users who have recordings on their phone, purchased music on the device, or who are transferring to a new Mac can copy music off the iPhone directly. The <a href="#iExplorer">iOS file system browsing</a> provides a tree-view of the iPhone's file system — navigating the folder structure of the device, browsing app data folders, accessing document directories, and viewing the complete file organization of the device that iOS and Finder don't expose through normal means. This file system access enables retrieving specific files from apps, accessing documents stored in app sandboxes, and understanding what data is actually on the device.

The <a href="#iExplorer">photos and videos transfer</a> copies media from the iPhone camera roll and photo library to the Mac — selectively downloading specific photos, videos, or date ranges rather than importing the entire photo library. <a href="#iExplorer">Voice memo and recording transfer</a> copies voice memos from the iPhone Voice Memos app to the Mac as audio files — retrieving recordings for use in audio workflows, transcription, or archiving without iCloud sync or AirDrop transfers.</p>

<a href="#iExplorer">Messages and notes browsing</a> reads iPhone text messages, iMessages, and notes — browsing message history, exporting conversations to PDF or text files, and accessing notes content from the Mac desktop. <a href="#iExplorer">App data access</a> browses the document directories of installed apps — accessing files stored by specific apps, retrieving documents from app sandboxes, and extracting data from apps that don't provide their own export mechanisms.

<a href="#iExplorer">Disk mode mounting</a> connects the iPhone as a disk volume on the Mac — mounting the device storage for direct file access through Finder, enabling drag-and-drop file transfer to and from the device as if it were an external drive. <a href="#iExplorer">iTunes backup browsing</a> reads existing iPhone backups stored on the Mac — extracting specific files, messages, or data from backups without restoring the entire backup to a device.</p>

iExplorer addresses the fundamental asymmetry in Apple's iOS-Mac file transfer design: iTunes and Finder support transferring content from Mac to iPhone but restrict the reverse. Music syncs one-way; files move under Apple's sync framework rather than as direct file access. This restriction becomes a real problem when a user's original sync Mac is no longer available, when they're setting up a new Mac and need to recover the music library from the phone, or when specific files on the device need retrieval that the normal sync mechanism doesn't support.
